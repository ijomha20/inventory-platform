# Code Review: Inventory Platform — Gaps & Flaws

## **CRITICAL SECURITY ISSUES**

### 1. **Hardcoded API Keys & Credentials in Client Code**
- **Location**: Apps Script, `lib/inventoryCache.ts`, `routes/inventory.ts`
- **Issue**: Typesense API keys embedded directly in source code
- **Risk**: Keys visible in browser DevTools, network requests, and version control
- **Impact**: Unauthorized database access, data exfiltration

### 2. **Session Secret Fallback**
- **Location**: `app.ts` line 31
- **Issue**: `SESSION_SECRET` defaults to `"dev-secret-change-me"`
- **Risk**: Production deployments without explicit env var use predictable secret
- **Impact**: Session hijacking, authentication bypass

### 3. **Missing CSRF Protection**
- **Location**: All POST/PATCH/DELETE routes
- **Issue**: No CSRF tokens on state-changing operations
- **Risk**: Cross-site request forgery attacks
- **Impact**: Unauthorized access grants, data modification

### 4. **Credential Exposure in Logs**
- **Location**: `lib/blackBookWorker.ts`, `lib/carfaxWorker.ts`
- **Issue**: Browser automation may log credentials during login flows
- **Risk**: Credentials in log aggregation systems
- **Impact**: Third-party service account compromise

### 5. **Unvalidated Webhook Secret**
- **Location**: `routes/inventory.ts` `/refresh` endpoint
- **Issue**: String comparison without timing-safe equality
- **Risk**: Timing attacks to brute-force `REFRESH_SECRET`
- **Impact**: Unauthorized cache poisoning

---

## **AUTHENTICATION & AUTHORIZATION FLAWS**

### 6. **Race Condition in Access Control**
- **Location**: `routes/inventory.ts` `requireAccess` middleware
- **Issue**: User role fetched per-request without transaction isolation
- **Risk**: TOCTOU (time-of-check/time-of-use) between role check and data access
- **Impact**: Privilege escalation during role changes

### 7. **No Session Expiration on Role Revocation**
- **Location**: `routes/access.ts` DELETE `/access/:email`
- **Issue**: Removing user from access list doesn't invalidate active sessions
- **Risk**: Revoked users retain access until cookie expires (7 days)
- **Impact**: Unauthorized data access post-termination

### 8. **Owner Email Case Sensitivity**
- **Location**: `lib/auth.ts` `isOwner()` function
- **Issue**: Comparison uses `.toLowerCase()` but env var may have mixed case
- **Risk**: Owner lockout if `OWNER_EMAIL` casing doesn't match Google profile
- **Impact**: Loss of admin access

### 9. **Missing Rate Limiting on Auth Endpoints**
- **Location**: `routes/auth.ts`
- **Issue**: `/auth/google` and `/auth/logout` excluded from rate limiter
- **Risk**: OAuth flow abuse, session exhaustion attacks
- **Impact**: Service degradation, quota exhaustion

---

## **DATA INTEGRITY & CONSISTENCY ISSUES**

### 10. **No Transaction Wrapping for Multi-Step Operations**
- **Location**: `routes/access.ts` POST `/access`
- **Issue**: DB insert + audit log write not atomic
- **Risk**: Audit log missing entries if second write fails
- **Impact**: Compliance violations, incomplete audit trail

### 11. **Cache Staleness Acceptance**
- **Location**: `lib/inventoryCache.ts` `refreshCache()`
- **Issue**: Empty upstream response keeps stale cache indefinitely
- **Risk**: Users see outdated inventory for hours/days
- **Impact**: Business decisions on incorrect data

### 12. **Black Book Value Merge Without Conflict Resolution**
- **Location**: `lib/inventoryCache.ts` `applyBlackBookValues()`
- **Issue**: Object storage values overwrite DB values without timestamp comparison
- **Risk**: Newer values replaced by stale cached data
- **Impact**: Incorrect wholesale pricing

### 13. **VIN Deduplication Logic Flaw**
- **Location**: `pages/inventory.tsx` (not shown in excerpt)
- **Issue**: "Keeps lowest-priced entry" — but price may be empty string
- **Risk**: Wrong record selected when comparing `""` vs `"25000"`
- **Impact**: Missing vehicle details in UI

## **ERROR HANDLING & RESILIENCE GAPS**

### 14. **Unhandled Promise Rejections**
- **Location**: `index.ts` startup sequence
- **Issue**: `scheduleCarfaxWorker()` and `scheduleBlackBookWorker()` not awaited
- **Risk**: Worker initialization errors silently ignored
- **Impact**: Automation never runs, no alerts

### 15. **Puppeteer Zombie Processes**
- **Location**: `lib/blackBookWorker.ts`, `lib/carfaxWorker.ts`
- **Issue**: Browser instances not closed in all error paths
- **Risk**: Memory leaks, process table exhaustion
- **Impact**: Server crash after repeated failures

### 16. **No Circuit Breaker for External APIs**
- **Location**: Typesense, NHTSA, CBB API calls
- **Issue**: Unlimited retries on persistent failures
- **Risk**: Cascading failures, request queue buildup
- **Impact**: Service outage

### 17. **Apps Script Error Suppression**
- **Location**: `InventorySync_FINAL.gs` `appendLog()` try/catch
- **Issue**: All logging errors silently swallowed
- **Risk**: Critical failures invisible to operators
- **Impact**: Data sync issues undetected for days

## **PERFORMANCE & SCALABILITY ISSUES**

### 18. **N+1 Query Pattern in Typesense Enrichment**
- **Location**: `lib/inventoryCache.ts` `fetchFromTypesense()`
- **Issue**: Sequential pagination requests (250 items/page)
- **Risk**: 10+ second cache refresh for 2000+ vehicles
- **Impact**: Slow API responses, timeout errors

### 19. **Unbounded Memory Growth in Cache**
- **Location**: `lib/inventoryCache.ts` `state.data` array
- **Issue**: No maximum size limit on in-memory cache
- **Risk**: OOM crash with 10,000+ vehicle inventory
- **Impact**: Service downtime

### 20. **Synchronous File I/O in Request Path**
- **Location**: `lib/bbObjectStore.ts` (implied by GCS SDK usage)
- **Issue**: Blocking calls during inventory fetch
- **Risk**: Request latency spikes during storage operations
- **Impact**: Poor user experience

### 21. **Excessive Polling Frequency**
- **Location**: `pages/inventory.tsx` 60-second cache status poll
- **Issue**: All connected users poll simultaneously
- **Risk**: Thundering herd on `/cache-status` endpoint
- **Impact**: Rate limit exhaustion, API overload

## **BUSINESS LOGIC FLAWS**

### 22. **Price Change Detection False Positives**
- **Location**: Apps Script `performSync()` price comparison
- **Issue**: Compares `parseFloat()` results without epsilon tolerance
- **Risk**: Floating-point precision errors trigger spurious alerts
- **Impact**: Email notification spam

### 23. **Trim Matching Score Ties**
- **Location**: `lib/blackBookWorker.ts` trim scoring algorithm
- **Issue**: Multiple trims with identical scores — first match wins
- **Risk**: Arbitrary selection between "LT" and "LTZ" trims
- **Impact**: Wrong wholesale value (up to $5,000 error)

### 24. **Carfax "NOT FOUND" Permanent Marking**
- **Location**: Apps Script `doGet()` filters out `"NOT FOUND"`
- **Issue**: VINs marked NOT FOUND never rechecked (even if added later)
- **Risk**: Missing Carfax links for newly available reports
- **Impact**: Compliance gaps, lost sales opportunities

### 25. **No Validation of VIN Format**
- **Location**: All VIN input paths
- **Issue**: Accepts any string as VIN (no checksum validation)
- **Risk**: API calls with invalid VINs waste quota
- **Impact**: NHTSA/CBB rate limit exhaustion
